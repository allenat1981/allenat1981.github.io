若要將目錄的檔案資料進行同步，可使用 rsync 指令。並且適用於 PHP 的網站部署。

## rsync 指令

若要將目錄 ProjectA 同步到 ProjectB

```bash
rsync -avz \
--delete \
--exclude='.git' \
--exclude='node_modules' \
--exclude='.env' \
./ProjectA/ ./ProjectB/
```

參數說明：

- -a (archive)：歸檔模式。它會保留檔案的權限、修改時間、軟連結等，是備份與部署的首選。

- -v (verbose)：顯示傳輸過程，讓你知道哪些檔案被更新了。

- -z (compress)：在傳輸過程中進行壓縮，節省頻寬（對文字檔案如 PHP/CSS 非常有效）。

- --delete：非常重要！ 如果你在本地刪除了一個檔案，遠端也會跟著刪除。這能確保遠端目錄與本地完全一致，不會殘留舊的垃圾檔案。

- --exclude：排除不需要上傳的目錄（如 Git 紀錄、前端開發套件、或是機密的 .env 設定檔）。

### 搭配 ssh 部署到遠端主機

若要指定遠端的 ssh port 或金鑰，則使用 `-e` 參數。  
-e 參數（全名是 --rsh）的作用是 「指定要使用的遠端 shell 程式」。  
它告訴 rsync：「請透過這個特定的通道（通常是 SSH）來連線到遠端主機。」

```bash
# -p 2222 指定遠端 port 2222
# -i 指定金鑰路徑

rsync -avz \
-e "ssh -p 2222 -i /path/to/private_key" \
--delete \
./ProjectA/ user@remote-host:/ProjectB/

# 若在自動化部署的情境下，要避免出現是否信任此主機的詢問導致卡住，可加上 -o StrictHostKeyChecking=no
rsync -avz \
-e "ssh -p 2222 -i /path/to/private_key -o StrictHostKeyChecking=no" \
--delete \
./ProjectA/ user@remote-server-ip:/ProjectB/
```

若有在 ~/.ssh/config 設定 Host，例如 `Host production-server ...`，則可用以下更簡單的指令

```bash
rsync -avz \
--delete \
./ProjectA/ product-server:~/ProjectB/
```

{{< alert type="notice" >}}
若是使用自動化部署，通常自動化工具的環境不會包含 ~/.ssh/config，此時建議使用 `-e` 進行設定。
{{< /alert >}}

### 權限問題

若部署後發生網頁伺服器沒權限讀取新檔案，可以在 rsync 指令中直接指定檔案權限：

```bash
rsync -avz \
--chmod=Du=rwx,Dg=rx,Do=rx,Fu=rw,Fg=r,Fo=r \
./ProjectA/ user@remote-host:~/ProjectB/
```

## rsync 比對策略

rsync 透過比對來源端與目的端的檔案，將差異的檔案進行重新傳輸，以達成同步化，以下為比對策略的簡略說明。

### 快速檢查

rsync 預設的比對方式，會先抓取來源端（Source）與目的端（Destination）檔案的中繼資料 (Metadata) 進行比對。

只要符合以下任一條件，rsync 就會認為檔案「沒變」，直接跳過：

- 檔案大小 (File Size) 一模一樣。

- 最後修改時間 (Modification Time) 一模一樣。

{{< alert type="warning" title="特別注意" >}}
如果檔案大小和時間都沒變，但內容被「刻意」改掉了，預設的快速檢查會漏掉它(視為相同)。  
若擔心發生此問題，可以加上 -c 參數強迫進行全檔案 Checksum (校驗碼) 比對。
{{< /alert >}}

### 調整比對策略

可以透過不同的參數調整比對策略

|參數|名稱|比對行為說明|
|---|---|----------|
|(預設)|Quick Check|只比對 檔案大小 + 修改時間。速度最快。|
|-c, --checksum|Checksum|改用 MD5/SHA1 雜湊值 比對全文內容。最精準但極耗 CPU。|
|-u, --update|Update|只有當來源端的檔案比目的端「更新」時才覆蓋。目的端的檔案修改時間（mtime）比來源端還要新，rsync 就會跳過這個檔案，不進行覆蓋。|
|-I, --ignore-times|Ignore Times|強迫不檢查時間，直接進行內容比對。|
|--size-only|Size Only|只要檔案大小一樣就當作沒變（無視修改時間）。|

{{< alert type="notice" >}}
除非有特殊狀況，否則一般而言使用預設 Quick Check 就已足夠。
{{< /alert >}}
