記錄使用 Laravel 12 搭配 pest 測試時的一些筆記

## 測試環境 env 檔

使用 pest 測試時，可以建立一個 .env.testing 檔案作為測試環境的測試檔。在測試檔內可以設定和 .env 一樣的內容，也可以僅設定測試需要的一些項目，例如建立一個測試的 mysql 或 mariadb 資料庫。

```conf
APP_KEY=測試環境 APP KEY
# 建立一個測試的資料庫，專門提供測試
DB_CONNECTION=mariadb
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=資料庫名稱
DB_USERNAME=資料庫帳號
DB_PASSWORD=資料庫密碼

# Custom 的自訂參數
DASHBOARD_PREFIX=dashboard

VITE_APP_NAME="${APP_NAME}"
VITE_ADMIN_BASE_URL="${APP_URL}/${DASHBOARD_PREFIX}"
```

{{< alert type="notice" >}}
功能測試時，可能會需要測試的 APP KEY，此時可以使用 `php artisan key:generate --env=testing` 產生。
{{< /alert >}}

當使用 .env.testing 設定的測試資料庫，可能會被 phpunit.xml 的設定覆蓋，此時需要將 phpunit.xml 內的資料庫設定取消或註解。

```xml
<!-- 找到 DB_CONNECTION 和 DB_DATABASE 並將其註解 -->
    <php>
        <env name="APP_ENV" value="testing"/>
        <env name="APP_MAINTENANCE_DRIVER" value="file"/>
        <env name="BCRYPT_ROUNDS" value="4"/>
        <env name="CACHE_STORE" value="array"/>
        <!-- <env name="DB_CONNECTION" value="sqlite"/>
        <env name="DB_DATABASE" value=":memory:"/> -->
        <env name="MAIL_MAILER" value="array"/>
        <env name="PULSE_ENABLED" value="false"/>
        <env name="QUEUE_CONNECTION" value="sync"/>
        <env name="SESSION_DRIVER" value="array"/>
        <env name="TELESCOPE_ENABLED" value="false"/>
    </php>
```

### 測試用的資料庫 seeder

功能測試時，若需要在測試資料庫預先使用 seeder 將測試資料載入，有以下 2 種方法。

#### 方法一:手動 migrate 與 seed

適合在測試前需要一次性產生資料庫與測試資料時，手動下 `artisan migrate` 指令配合 .env.testing 檔案將整個測試資料庫建立起來，並搭配 `Illuminate\Foundation\Testing\DatabaseTransactions::class` 用 Transaction + Rollback 機制，每次測試完將資料庫 Rollback 為初始狀態。好處是整個測試過程只需要建立一次資料庫結構和 seed 一次。

首先編輯 tests/Pest.php

```php
pest()->extend(Tests\TestCase::class)
    ->use(Illuminate\Foundation\Testing\DatabaseTransactions::class)
    ->in('Feature');
```

以上設定說明

- `extend(Tests\TestCase::class)` 代表 pest 每個測試都會繼承 Tests\TestCase 類別。
- `use(Illuminate\Foundation\Testing\DatabaseTransactions::class)` 代表在每個測試案例開始和結束時分別調用 DatabaseTransactions::class 的 Transaction 和 Rollback 機制，來達成每次測試結束時，資料庫仍可保持初始狀態。
- `in('Feature')` 代表上述設定只針對 tests/Feature 目錄內的檔案生效。

接著建立測試資料庫

```bash
# 測試前先使用 migrate 指令建立整個測試資料庫
php artisan migrate:fresh --seed --env=testing
```

接著即可進行測試

```bash
# 若要測試整個專案(單元測試 + 功能測試)
php artisan test

# 若僅僅要功能測試
php artisan test --testsuite=Feature

```

#### 方法二:使用 RefreshDatabase 每次測試時自動清空並建立資料庫與測試資料

適用於比較小的資料量。因為每次執行測試時，都會 migrate:fresh 並且 seed 資料，所以當專案資料庫較大時，速度較慢。好處是可以確保每次測試時，資料庫都會重新初始化一次。

首先編輯 tests/Pest.php

```php
pest()->extend(Tests\TestCase::class)
    ->use(Illuminate\Foundation\Testing\RefreshDatabase::class)
    ->in('Feature');
```

與方法一不同的地方在於，此方法是使用 RefreshDatabase 替代 DatabaseTransactions 機制。

接著編輯 tests/TestCase.php

```php
abstract class TestCase extends BaseTestCase
{
    // 加入 $seed = true
    public bool $seed = true;
}
```

接著就可以直接進行測試。

方法一和方法二的差別在於  
方法一：手動 migrate 一次後，後續呼叫 `php artisan test` 都不需要等待 migrate:fresh 和 seed 的時間。
方法二：每次 `php artisan test` 時，都需要等待一次 migrate:fresh。

若進行 10 次 `php artisan test`  
方法一：在需要的時候，手動跑 1 次 migrate。
方法二：不需手動 migrate ，但每次執行都會自動 1 次 migrate(共 10 次)。
